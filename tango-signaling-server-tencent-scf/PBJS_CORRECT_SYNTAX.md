# PBJS Correct Command Syntax

## Error You Saw

```
TypeError [ERR_INVALID_ARG_TYPE]: The "path" argument must be of type string...
Received undefined at Object.readFileSync (node:fs:438:14)
```

## Root Cause

The `pbjs` command syntax was incorrect. The tool expects:
```bash
pbjs --es6 OUTPUT_FILE INPUT_PROTO_FILE
```

But we were trying:
```bash
pbjs --es6 INPUT_PROTO_FILE > OUTPUT_FILE  # WRONG - shell redirection doesn't work
```

## The Fix (Applied)

Updated `package.json` to use correct argument order:

**Before (Wrong):**
```json
"proto": "npx pbjs --es6 ../tango-signaling/src/proto/signaling.proto > src/proto/signaling.js"
```

**After (Correct):**
```json
"proto": "npx pbjs --es6 src/proto/signaling.js ../tango-signaling/src/proto/signaling.proto"
```

### What Changed:
1. Output file FIRST: `src/proto/signaling.js`
2. Input file SECOND: `../tango-signaling/src/proto/signaling.proto`
3. Removed shell redirection `>` (doesn't work with pbjs)

## Correct pbjs Command Format

```bash
pbjs --es6 <output.js> <input.proto>
          │             │
          └─ goes first  └─ goes second
```

## Full Proto Script

```json
"proto": "mkdir -p src/proto && npx pbjs --es6 src/proto/signaling.js ../tango-signaling/src/proto/signaling.proto && npx pbts src/proto/signaling.d.ts src/proto/signaling.js"
```

This does:
1. Create `src/proto/` directory
2. Generate `signaling.js` from `signaling.proto`
3. Generate `signaling.d.ts` from `signaling.js`

## Command Reference

### pbjs Commands

**Generate ES6 JavaScript:**
```bash
pbjs --es6 output.js input.proto
```

**Generate ES5 JavaScript:**
```bash
pbjs --es5 output.js input.proto
```

**Generate TypeScript (type definitions only):**
```bash
pbjs --ts output.ts input.proto
```

### pbts Command

**Generate TypeScript definitions:**
```bash
pbts output.d.ts input.js
```

## Common Mistakes to Avoid

### ❌ Wrong: Shell redirection
```bash
pbjs --es6 input.proto > output.js  # WRONG
```

### ❌ Wrong: -o flag
```bash
pbjs --es6 -o output.js input.proto  # WRONG
```

### ❌ Wrong: Output file second
```bash
pbjs --es6 input.proto output.js  # WRONG
```

### ✓ Correct: Output file first
```bash
pbjs --es6 output.js input.proto  # CORRECT
```
